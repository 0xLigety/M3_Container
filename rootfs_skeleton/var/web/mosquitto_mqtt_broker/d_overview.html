Mosquitto MQTT-Broker

Dies ist ein Demo-Container für die SmartBox der MRX-Router. Er wurde von INSYS icom auf der Basis von verfügbarer Software von Dritten insbesondere dem MQTT-Broker Mosquitto erstellt. MQTT ist eines der führenden Protokolle für die Kommunikation im IoT (Internet of Things), da seine Publisher-Subscriber-Architektur die notwendige Informationsübermittlung ohne Polling schlank und zuverlässig erledigt. Der Broker nimmt in der MQTT-Architektur Daten von Publishern zu einem Topic entgegen und leitet sie an die Subscriber weiter, die sich dafür angemeldet haben.

Publisher und Subscriber müssen TCP/IP-Verbindungen zum Broker aufbauen. Die Netzwerk-Fähigkeiten des Routers (z.B. OpenVPN über INSYS Connectivity Service) können den Verbindungsaufbau unterstützen. Zum Betrieb dieser Anwendung brauchen Sie:
  • einen MQTT-Publisher (z.B. die Monitoring App von INSYS-OS-Routern oder der Demo-Container Modbus-MQTT-Gateway für die MRX-SmartBox) als Quelle
  • ein MQTT-Subscriber-Programm auf einem PC oder Smartphone (für Empfehlungen siehe HiveMQ/Clients) zur Anzeige der Werte

Konfiguration

  • Die Bind-Adresse dient zum Zugriffsschutz und bleibt in der Regel für Demonstrationszwecke leer.
  • Der TCP-Port gibt vor, wohin sich Publisher und Subscriber verbinden. Default-Ports sind 1883 und 8883 (für TLS).
  • Die IP-Adresse des Containers selbst als Ziel für die Verbindungen vom Publisher und vom Subscriber wird im Menü Administration/Container des Routers verwaltet
Hinweis: Konfigurationsänderungen werden erst nach Betätigen der Schaltfläche Einstellungen aktivieren in der Kopfzeile wirksam.

Demonstration Container

INSYS icom stellt diesen Demo-Container ohne Zusicherung einer Funktion bereit. Er wird von INSYS icom nicht weiter gepflegt. Dieser Container benutzt Software von Dritten, deren Rechte berücksichtig werden müssen. Dieser Demo-Container ist nur dazu erstellt worden, die Verwendung von bestehenden Technologien in der SmartBox der MRX-Router zu zeigen. Dieser Demo-Container ist nicht zum Betrieb au&erhalb von Testumgebungen freigegeben. Der Einblick in diesen Container für weitere Untersuchungen und Anpassungen wird auf zwei Arten gewährt:
  • Auf die Kommandozeile kann mit dem Benutzernamen root und dem Passwort root per ssh zugegriffen werden. Damit kann der Container im laufenden Betrieb beobachtet und verändert werden.
  • Der Container mit allen verfügbaren Quellen wird auf den Projektseiten auf Github veröffentlicht.
