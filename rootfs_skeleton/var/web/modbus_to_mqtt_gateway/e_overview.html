Modbus to MQTT Gateway

This is a demonstration container running in the SmartBox of MRX routers. It is provided by INSYS icom on the base of extisting 3rd party software like
  • a Modbus library (libmodbus)
  • an MQTT publisher (Mosuqitto)
and combines them in an application draft which
  • periodically reads 1 register froma ModbusTCP device
  • uploads the value of the register as a user defines topic to an MQTT broker.

The routers network capabilities can be used to connect to the MQTT broker. MQTT is one of the leading protocols for the IoT (Internet of Things) as its publisher-subscriber architecture realises the necessary information flow without polling in slim and reliable way.
To operate this showcase you need

  • a ModbusTCP device (e.g. a PLC) as a data source
  • access to a MQTT broker (e.g. the free HiveMQ or ademonstrator from INSYS in another container of your MRX router)
  • a MQTT subscriber application running on a PC or smartphone (see recommendations on HiveMQ/Clients) to display the values

Configuration

  • The access to the ModbusTCP device and to the MQTT broker is configured in the menu "Application"
  • Set the IP address of the MRX router as default gateway and DNS server in the IP network menu
  • The IP address of the container itself is managed in the routerÂ´s menu Administration/Container

Additional MQTT options such as QoS, TLS, user/password authentication, Birth Message, Last Will are supported by Mosquitto, but not available with this demonstrator.
Beware: Changes of the configuration take effect only after clicking on activate settings in the headline.
